## 关键词

## 摘要

促进数字经济（DE）与城乡一体化（URI）的协调发展被认为是实现可持续增长的关键路径。本文提出了一种数据驱动的方法，用于测量、识别和测试DE与URI之间的耦合协调度（DE-URI CCD）。首先，收集了DE和URI的相关数据，并建立了评估指标体系。随后，应用TOPSIS方法测量发展水平，并使用CCD模型评估其协调性。同时，应用Dagum Gini系数识别区域差异，并使用收敛模型验证水平的收敛特征。基于2013-2022年中国30个省市的面板数据，研究结果表明：（1）中国DE-URI CCD呈上升趋势，但区域间存在显著差异；（2）区域间差异是区域CCD差异的主要来源；（3）经济发展水平和劳动力教育水平对CCD的影响各异，表现出显著的区域异质性。为了解决这些挑战，我们提出了一个多层次的“国家-区域-省级”协同策略，强调需要根据区域特殊性制定定制政策。这项研究不仅丰富了可持续区域发展的理论和方法基础，还为旨在弥合数字时代城乡鸿沟的政策制定者提供了可操作的见解。通过与联合国可持续发展目标（SDGs）保持一致，特别是SDG 11（可持续城市和社区），这项工作强调了将数字创新与包容性发展框架整合的重要性。

## 研究思路

### 为什么要做这个研究

#### 现实重要性

论文指出的现实问题是数字经济快速发展与城乡差距持续存在的矛盾。数字经济已成为全球经济增长的关键催化剂，2022年中国数字经济规模达50.2万亿元人民币，占GDP的41.5%。然而，城乡发展不平衡问题突出，2023年中国城镇居民人均可支配收入是农村居民的2.37倍，数字技术在农业部门的渗透率仅10.5%，远低于服务业和工业。这些问题具有紧迫性，因为城乡数字鸿沟会加剧社会不平等、阻碍可持续发展，未解决可能导致经济增长放缓、社会不稳定和环境恶化。影响范围涵盖经济、社会和环境多个维度，涉及资源配置、收入分配和公共服务均等化。

#### 政策重要性

论文提到中国2018年中央一号文件首次提出实施数字乡村战略，系统推进农业农村数字化转型。政策背景包括中国的乡村振兴战略、区域协调发展战略等。研究对政策制定有意义，可为数字乡村建设、城乡融合政策提供量化依据，帮助评估政策效果。研究结果的政策含义在于需要差异化策略，例如在东部地区强调数字技术溢出，在西部地区加强基础设施投资。研究有助于实现联合国可持续发展目标（SDG 11），为全球可持续发展提供参考。

### 1.1 数据来源与获取

- •
    
    数据来源：中国统计年鉴、中国环境统计年鉴、中国农村统计年鉴、中国城乡建设统计年鉴、各省统计年鉴、北京大学数字普惠金融指数报告（公开数据）。
    
- •
    
    数据公开性：□ 公开 □ 非公开 □ 部分公开（所有数据均来自公开来源，获取时间为2013-2022年）。
    
- •
    
    原始数据特征：格式为面板数据，涵盖30个中国省份，时间跨度为10年（2013-2022），字段包括数字经济指标（15个）和城乡一体化指标（22个），样本展示如数字基础设施指标（移动电话普及率、互联网域名数等）、城乡要素整合指标（城乡人口比例、农业机械化水平等）。
    

### 1.2 数据预处理流程

- •
    
    **操作目的**：消除量纲影响，客观赋权。
    
- •
    
    **具体操作**：使用极差标准化法对数据进行归一化处理，公式为：对于正向指标，x′=(x−min)/(max−min)；对于负向指标，x′=(max−x)/(max−min)。随后，对每个值加0.0001以避免零值，计算指标比例、信息熵和冗余度，最终得出指标权重。
    
- •
    
    **输入输出**：输入为原始数据矩阵，输出为标准化后的加权数据矩阵。
    
- •
    
    **验证方法**：检查权重之和是否为1，并验证标准化后数据范围在[0,1]之间。
    
- •
    
    **预期结果**：权重反映指标离散程度，熵值小的指标权重高。
    

数据清理清单：

- •
    
    缺失值：采用线性插值法填补，如个别年份数据缺失。
    
- •
    
    异常值：使用3σ原则识别并修正，如超出均值±3倍标准差的值。
    
- •
    
    重复数据：检查并删除重复条目。
    
- •
    
    数据转换：所有指标转换为正向指标，确保方向一致性。
    
- •
    
    数据合并：将数字经济与城乡一体化数据按省份和年份合并为面板数据。
    

### 1.3 最终数据集

- •
    
    数据集描述：包含30个省份、10年、37个指标的面板数据，样本量300。
    
- •
    
    统计摘要：例如，数字经济指数均值0.35（2013年）至0.45（2022年），城乡一体化指数均值0.30至0.40。
    
- •
    
    保存格式：CSV或Excel格式，包含省份、年份、指标值字段。
    
- •
    
    质量检查：通过描述性统计（均值、标准差）验证数据分布，确保无缺失或异常。
    

## 2. 方法部分

### 2.1 软件环境与依赖

- •
    
    软件/工具清单：Python 3.8或R 4.0；依赖包：pandas用于数据处理，numpy用于数值计算，scikit-learn用于TOPSIS实现；分析工具：Excel用于初步整理；操作系统：Windows/Linux。
    
- •
    
    环境配置：安装Python和所需包（例如，pip install pandas numpy scikit-learn），验证步骤为运行简单测试脚本检查包导入是否成功。
    

### 2.2 理论方法与公式

- •
    
    方法概述：使用熵权TOPSIS法测量发展水平，CCD模型计算协调度，Dagum Gini系数分析区域差异，收敛模型验证动态趋势。
    
- •
    
    数学公式：
    
    - •
        
        熵权法：信息熵 Ej​=−∑i=1n​pij​lnpij​，权重 wj​=(1−Ej​)/∑j=1m​(1−Ej​)。
        
    - •
        
        TOPSIS法：正理想解 S+=∑(xij​−xj+​)2​，负理想解 S−=∑(xij​−xj−​)2​，贴近度 Ci​=S−/(S++S−)。
        
    - •
        
        CCD模型：耦合度 C=(F1​×F2​)/[(F1​+F2​)/2]2​，协调指数 T=α1​F1​+α2​F2​，CCD值 Y=C×T​。
        
    - •
        
        Dagum Gini系数：总体基尼系数 G=∑e=1n​∑f=1n​∑g=1me​​∑h=1mf​​∣yeg​−yfh​∣/(2m2yˉ​)。
        
    - •
        
        收敛模型：σ收敛用变异系数 CV=σ/μ，β收敛用公式 ln(Yi,t+1​/Yi,t​)=α+βlnYi,t​+∑θk​Xk,i,t​+ϵi,t​。
        
    
- •
    
    符号含义：F1​和 F2​分别代表DE和URI的发展水平，α1​和 α2​为权重（均设0.5），yeg​为区域e省份g的CCD值。
    
- •
    
    方法假设：数据服从正态分布，系统间相互作用线性可加。
    
- •
    
    选择理由：熵权TOPSIS客观赋权，CCD模型适合系统协调分析，Dagum Gini系数能分解差异来源，收敛模型捕捉动态趋势。
    

### 2.3 实现步骤（原子级分解）

**步骤1: 数据标准化**

- •
    
    步骤目的：统一指标量纲，便于加权。
    
- •
    
    输入数据：原始指标数据矩阵。
    
- •
    
    具体操作：对每个指标应用极差标准化公式，计算归一化值。
    
- •
    
    核心变量测算：例如，计算移动电话普及率的归一化值，公式为 x′=(x−0)/(100−0)（假设范围0-100），示例：某省值50，则 x′=0.5。
    
- •
    
    参数设置：无参数。
    
- •
    
    中间输出：标准化数据矩阵。
    
- •
    
    验证检查：检查数据是否在[0,1]范围内。
    

**步骤2: 熵权法计算权重**

- •
    
    步骤目的：客观分配指标权重。
    
- •
    
    输入数据：标准化后的数据矩阵。
    
- •
    
    具体操作：计算每个指标的比重 pij​=xij​/∑xij​，信息熵 Ej​=−∑pij​lnpij​，权重 wj​=(1−Ej​)/∑(1−Ej​)。
    
- •
    
    核心变量测算：例如，数字基础设施维度下，计算各指标权重，假设信息熵E_j=0.8，则权重 wj​=(1−0.8)/∑(1−Ej​)。
    
- •
    
    参数设置：无参数。
    
- •
    
    中间输出：各指标权重向量。
    
- •
    
    验证检查：权重之和为1。
    

**步骤3: TOPSIS法计算发展水平**

- •
    
    步骤目的：测量DE和URI的发展水平。
    
- •
    
    输入数据：加权标准化数据（标准化数据×权重）。
    
- •
    
    具体操作：确定正负理想解，计算欧氏距离，得出贴近度。
    
- •
    
    核心变量测算：例如，DE发展水平 F1​=Ci​，计算过程：正理想解为各指标最大值，负理想解为最小值，距离 S+=∑(xij​−max)2​，S−=∑(xij​−min)2​，Ci​=S−/(S++S−)。
    
- •
    
    参数设置：无参数。
    
- •
    
    中间输出：DE和URI的发展水平值。
    
- •
    
    验证检查：贴近度应在[0,1]之间。
    

**步骤4: CCD模型计算协调度**

- •
    
    步骤目的：评估DE与URI的耦合协调度。
    
- •
    
    输入数据：DE和URI的发展水平值 F1​和 F2​。
    
- •
    
    具体操作：计算耦合度C、协调指数T，最终CCD值Y。
    
- •
    
    核心变量测算：例如，C=(0.4×0.3)/[(0.4+0.3)/2]2​=0.12/0.1225​≈0.99，T=0.5×0.4+0.5×0.3=0.35，Y=0.99×0.35​≈0.59。
    
- •
    
    参数设置：权重 α1​=α2​=0.5。
    
- •
    
    中间输出：CCD值。
    
- •
    
    验证检查：Y值应在[0,1]之间，分类参考论文图3。
    

**步骤5: Dagum Gini系数分析差异**

- •
    
    步骤目的：分解区域差异来源。
    
- •
    
    输入数据：各省CCD值，区域划分（东、中、西）。
    
- •
    
    具体操作：计算总体基尼系数、区域内差异、区域间差异、超变密度。
    
- •
    
    核心变量测算：例如，总体G值，公式为 G=∑∣yi​−yj​∣/(2m2yˉ​)，示例计算：假设m=30，yˉ​=0.3，差异和100，则 G=100/(2×900×0.3)≈0.185。
    
- •
    
    参数设置：区域划分依据中国统计年鉴。
    
- •
    
    中间输出：差异分解结果。
    
- •
    
    验证检查：G值在[0,1]之间，贡献率之和为100%。
    

**步骤6: 收敛模型验证**

- •
    
    步骤目的：检验CCD的收敛特征。
    
- •
    
    输入数据：CCD值面板数据。
    
- •
    
    具体操作：计算σ收敛（变异系数），β收敛（回归分析），空间自相关检验（Moran's I）。
    
- •
    
    核心变量测算：例如，σ收敛 CV=σ/μ，β收敛系数β从回归得出，收敛速度 s=−ln(1+β)/T。
    
- •
    
    参数设置：控制变量如TEC、OPEN等，数据来自年鉴。
    
- •
    
    中间输出：收敛系数和速度。
    
- •
    
    验证检查：β为负表示收敛，Moran's I显著为正表示空间相关。
